# SPEC-SERVER-RF-001 — 백엔드 리팩터링 & 품질 개선

## 목표
구현 완료된 백엔드를 **클린아키텍처 · 보안성 · 확장성 · 유지보수성** 4차원에서 점검하고, **동작 보존(behavior-preserving)** 리팩터를 적용한 뒤, 결과를 문서화한다. 기능 변경 없음 — 구조/품질만 개선.

## 절차 (HARD)
1. **AUDIT(점검)**: 전체 코드베이스를 4차원으로 점검해 리팩터 후보를 **우선순위(높음/중간/낮음)와 함께 목록화** → `.moai/specs/SPEC-SERVER-RF-001/audit.md`(한국어)에 기록.
2. **REFACTOR(실행)**: 후보를 우선순위 순으로 적용. **각 리팩터 단위마다 `./gradlew build test` 그린 유지**(깨지면 즉시 롤백). 작은 단위로 커밋.
3. **DOCUMENT(문서화)**: 아키텍처 문서(`docs/`) 갱신 + 리팩터 로그(`.moai/specs/SPEC-SERVER-RF-001/refactor-log.md` — 무엇을/왜/효과) 작성. 모두 한국어.

## 점검 차원 & 체크리스트
- **클린아키텍처**: controller→service→repository 경계 준수, DTO↔엔티티 분리(엔티티 외부 노출 0), 도메인 응집도, 순환 의존 제거, 공통 관심사(`common/`) 적절 분리.
- **보안 [최우선]**: **모든 데이터 쿼리에 `user_id` 격리 누락 0건 재검증**(멀티테넌시 = 유일 방어선), 입력 검증(Bean Validation), 인증/인가 일관성, 시크릿 하드코딩 0, 에러 응답 내부정보 비노출, SQL 인젝션(파라미터 바인딩), OWASP Top 10.
- **확장성**: 새 도메인 추가가 기존 코드 수정 없이 가능한지, 중복 로직 공통화, 인터페이스 추상화 **적정선**(과추상·미사용 추상 금지 = YAGNI).
- **유지보수성**: 네이밍 명확성, 함수/클래스 크기(과대 분해), 매직넘버 상수화, 테스트 커버리지 보강(핵심 로직), 코드 스타일 일관성(ktlint/detekt).

## 인수기준

- [ ] `refactor-log.md` + 아키텍처 문서 갱신(한국어)

## 비고
- 리팩터로 동작이 바뀌면 안 됨. 불확실하면 특성화 테스트 먼저 추가 후 리팩터.
- 범위는 구조/품질 개선까지. 새 기능·배포 없음.
