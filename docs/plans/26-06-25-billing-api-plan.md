# 토스 빌링 API 구현 계획 — Part 1: 기반(Foundation)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** flori 토스페이먼츠 자동결제(빌링) 백엔드의 기반 계층 — 6개 테이블/엔티티, 마이그레이션, 설정, 에러코드, 빌링키 암호화, 토스 HTTP 클라이언트 — 을 기존 컨벤션대로 구축한다.

**Architecture:** 신규 `kr.ai.flori.billing` 도메인. 엔티티는 `BaseEntity`/`BaseCreatedEntity` 상속, FK 미사용(간접참조), 멀티테넌시는 `user_id`로 격리. 토스 호출은 `RestClient` 기반 `BillingClient`로 캡슐화. billingKey는 앱레벨 AES 암호화 후 저장.

**Tech Stack:** Spring Boot 3.5 / Kotlin 2.1 / Java 21 / JPA(Hibernate, ddl-auto=validate) / PostgreSQL / RestClient / JUnit5 + Zonky 임베디드 PG + MockRestServiceServer.

## Global Constraints

- 플랜·금액: `MONTHLY` = **14900**원, `YEARLY` = **154800**원 (일시불). `amount`는 구독 생성 시 스냅샷(정수, KRW).
- 무료체험: **14일**. 신원(provider+providerId) 1개당 1회.
- dunning: 결제 실패 → `IN_GRACE`, 유예 **3일**, 매일 재시도, 유예 소진 시 `EXPIRED`.
- 구독 상태: `TRIALING` / `ACTIVE` / `IN_GRACE` / `EXPIRED` (구독 없음 = row 없음).
- 타임존: 모든 cron·날짜 계산 `Asia/Seoul`(KST). 시각 컬럼은 `TIMESTAMPTZ`/`Instant`.
- DB 컨벤션: PK `BIGINT GENERATED BY DEFAULT AS IDENTITY`, snake_case, **FK 제약 미사용(간접참조)**, `ddl-auto=validate`. 새 테이블은 `docs/sql/migration/26-06-25-billing.sql` **및** `docs/sql/all-tables-ddl.sql`(테스트가 로드)에 모두 반영.
- 에러코드: `E-BILLING-NNN`, `ErrorCode` 인터페이스 구현 enum.
- 보안: `billing_key`는 **AES 가역 암호화**(단방향 해시 금지 — 매 결제 시 원문 필요). `identity_hash`는 **SHA-256 단방향**.
- 코드스타일: ktlint + detekt 통과 필수. 커버리지: JaCoCo LINE 80%(`check` 게이트).
- 커밋: `feat(billing): ...` / `Co-Authored-By: Claude <noreply@anthropic.com>`. `git add`는 변경 파일만 명시.

-- 토스페이먼츠 자동결제(빌링) + 구독 + 쿠폰.
-- 컨벤션: BIGINT IDENTITY PK, snake_case, FK 미사용(간접참조), updated_at 트리거(update_updated_at()).
-- billing_key.billing_key 는 앱레벨 AES 암호문(평문 금지). identity_hash 는 SHA-256.
START TRANSACTION;

-- 1) billing_key: 유저당 카드 1장(토스 빌링키 보관함)
CREATE TABLE billing_key (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id BIGINT NOT NULL,
  customer_key VARCHAR(64) NOT NULL,             -- 우리가 생성한 토스 customerKey(UUID)
  billing_key TEXT NOT NULL,                     -- AES 암호문(base64)
  card_company VARCHAR(40),
  card_number_masked VARCHAR(40),
  card_type VARCHAR(20),
  status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' CHECK (status IN ('ACTIVE','DELETED')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_billing_key_user ON billing_key(user_id);

-- 2) subscription: 유저당 구독 1건
CREATE TABLE subscription (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id BIGINT NOT NULL,
  plan VARCHAR(10) NOT NULL CHECK (plan IN ('MONTHLY','YEARLY')),
  status VARCHAR(12) NOT NULL CHECK (status IN ('TRIALING','ACTIVE','IN_GRACE','EXPIRED')),
  billing_key_id BIGINT,
  amount INTEGER NOT NULL,
  current_period_start TIMESTAMPTZ,
  current_period_end TIMESTAMPTZ,
  next_billing_at TIMESTAMPTZ NOT NULL,
  cancel_at_period_end BOOLEAN NOT NULL DEFAULT FALSE,
  grace_until TIMESTAMPTZ,
  retry_count INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_subscription_user ON subscription(user_id);
CREATE INDEX idx_subscription_due ON subscription(next_billing_at)
  WHERE status IN ('TRIALING','ACTIVE','IN_GRACE');

-- 3) payment_history: 결제 시도 장부(append-only)
CREATE TABLE payment_history (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id BIGINT NOT NULL,
  subscription_id BIGINT NOT NULL,
  order_id VARCHAR(64) NOT NULL,                 -- 우리 생성, 시도마다 유니크
  billing_cycle DATE NOT NULL,                   -- 이 결제가 속한 주기(이용기간 시작일)
  amount INTEGER NOT NULL,
  status VARCHAR(10) NOT NULL CHECK (status IN ('PAID','FAILED','CANCELED')),
  toss_payment_key VARCHAR(64),
  failure_code VARCHAR(60),
  failure_message TEXT,
  approved_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_payment_order ON payment_history(order_id);
-- 한 주기당 성공결제 1건만(이중청구 방지)
CREATE UNIQUE INDEX uq_payment_paid_per_cycle ON payment_history(subscription_id, billing_cycle)
  WHERE status = 'PAID';
CREATE INDEX idx_payment_subscription ON payment_history(subscription_id, created_at DESC);

-- 4) coupon: 쿠폰 정의(콘솔 발행)
CREATE TABLE coupon (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  code VARCHAR(40) NOT NULL,
  grant_type VARCHAR(20) NOT NULL DEFAULT 'FREE_DAYS' CHECK (grant_type IN ('FREE_DAYS')),
  days INTEGER NOT NULL,
  valid_from TIMESTAMPTZ,
  valid_until TIMESTAMPTZ,
  max_redemptions INTEGER,
  per_user_limit INTEGER NOT NULL DEFAULT 1,
  redeemed_count INTEGER NOT NULL DEFAULT 0,
  status VARCHAR(12) NOT NULL DEFAULT 'ACTIVE' CHECK (status IN ('ACTIVE','DISABLED')),
  source VARCHAR(20) NOT NULL DEFAULT 'PROMO' CHECK (source IN ('PROMO','REFERRAL','EVENT','MANUAL')),
  memo TEXT,
  created_by BIGINT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_coupon_code ON coupon(code);

-- 5) coupon_redemption: 쿠폰 사용 기록
CREATE TABLE coupon_redemption (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  coupon_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  granted_days INTEGER NOT NULL,
  subscription_id BIGINT,                        -- 가입 전 redeem이면 NULL(pending)
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_coupon_redemption_user ON coupon_redemption(coupon_id, user_id);
CREATE INDEX idx_coupon_redemption_coupon ON coupon_redemption(coupon_id, created_at DESC);

-- 6) subscription_eligibility: 어뷰징 방어 신원 원장(탈퇴해도 유지)
CREATE TABLE subscription_eligibility (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  identity_hash VARCHAR(64) NOT NULL,            -- SHA-256(provider:providerId)
  trial_used_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX uq_eligibility_identity ON subscription_eligibility(identity_hash);

-- updated_at 자동 갱신 트리거(기존 공용 함수 update_updated_at() 재사용)
CREATE TRIGGER trg_billing_key_updated BEFORE UPDATE ON billing_key
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER trg_subscription_updated BEFORE UPDATE ON subscription
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER trg_coupon_updated BEFORE UPDATE ON coupon
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER trg_eligibility_updated BEFORE UPDATE ON subscription_eligibility
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

COMMIT;
```

package kr.ai.flori.billing.crypto

import jakarta.persistence.AttributeConverter
import jakarta.persistence.Converter
import org.springframework.beans.factory.annotation.Value
import org.springframework.stereotype.Component
import java.security.SecureRandom
import java.util.Base64
import javax.crypto.Cipher
import javax.crypto.spec.GCMParameterSpec
import javax.crypto.spec.SecretKeySpec

/**
 * billingKey 컬럼 가역 암호화(AES-256-GCM). 단방향 해시 금지 — 매 결제 시 원문 필요.
 * 키는 BILLING_ENCRYPTION_KEY(32바이트의 base64) 환경변수에서 주입.
 * 저장형식: base64(IV(12B) || ciphertext+tag). IV는 매번 랜덤(같은 평문도 다른 암호문).
 *
 * Spring Boot가 @Component 컨버터를 Hibernate에 주입한다(스프링 관리 빈).
 */
@Component
@Converter
class BillingKeyCryptoConverter(
    @Value("\${billing.encryption-key}") encryptionKeyBase64: String,
) : AttributeConverter<String, String> {
    private val key = SecretKeySpec(Base64.getDecoder().decode(encryptionKeyBase64), "AES")
    private val random = SecureRandom()

    override fun convertToDatabaseColumn(attribute: String?): String? {
        if (attribute == null) return null
        val iv = ByteArray(IV_BYTES).also { random.nextBytes(it) }
        val cipher = Cipher.getInstance(TRANSFORMATION)
        cipher.init(Cipher.ENCRYPT_MODE, key, GCMParameterSpec(TAG_BITS, iv))
        val ct = cipher.doFinal(attribute.toByteArray(Charsets.UTF_8))
        return Base64.getEncoder().encodeToString(iv + ct)
    }

    override fun convertToEntityAttribute(dbData: String?): String? {
        if (dbData == null) return null
        val all = Base64.getDecoder().decode(dbData)
        val iv = all.copyOfRange(0, IV_BYTES)
        val ct = all.copyOfRange(IV_BYTES, all.size)
        val cipher = Cipher.getInstance(TRANSFORMATION)
        cipher.init(Cipher.DECRYPT_MODE, key, GCMParameterSpec(TAG_BITS, iv))
        return String(cipher.doFinal(ct), Charsets.UTF_8)
    }

    private companion object {
        const val TRANSFORMATION = "AES/GCM/NoPadding"
        const val IV_BYTES = 12
        const val TAG_BITS = 128
    }
}
```

```kotlin
package kr.ai.flori.billing.client

import com.fasterxml.jackson.annotation.JsonIgnoreProperties
import kr.ai.flori.billing.config.TossPaymentsProperties
import kr.ai.flori.billing.error.BillingErrorCode
import kr.ai.flori.common.error.AppException
import org.slf4j.LoggerFactory
import org.springframework.http.HttpHeaders
import org.springframework.http.MediaType
import org.springframework.stereotype.Component
import org.springframework.web.client.RestClient
import org.springframework.web.client.RestClientResponseException
import java.util.Base64

/** 토스페이먼츠 빌링 API 캡슐화. secretKey Basic 인증, 거절/오류는 AppException으로 wrap. */
@Component
class BillingClient(
    private val properties: TossPaymentsProperties,
    builder: RestClient.Builder,
) {
    private val log = LoggerFactory.getLogger(javaClass)
    private val restClient: RestClient = builder.build()
    private val authHeader: String =
        "Basic " + Base64.getEncoder().encodeToString("${properties.secretKey}:".toByteArray())

    fun issueBillingKey(authKey: String, customerKey: String): IssuedBilling {
        val res = post(
            "/v1/billing/authorizations/issue",
            mapOf("authKey" to authKey, "customerKey" to customerKey),
            null,
            IssueResponse::class.java,
            BillingErrorCode.BILLING_KEY_ISSUE_FAILED,
        )
        return IssuedBilling(res.billingKey, res.card?.company, res.card?.number, res.card?.cardType)
    }

    fun approveBilling(
        billingKey: String,
        customerKey: String,
        amount: Int,
        orderId: String,
        orderName: String,
        idempotencyKey: String,
    ): ApprovedPayment {
        val res = post(
            "/v1/billing/$billingKey",
            mapOf("customerKey" to customerKey, "amount" to amount, "orderId" to orderId, "orderName" to orderName),
            idempotencyKey,
            ApproveResponse::class.java,
            BillingErrorCode.PAYMENT_REJECTED,
        )
        return ApprovedPayment(res.paymentKey, res.orderId, res.approvedAt)
    }

    private fun <T> post(
        path: String,
        body: Map<String, Any>,
        idempotencyKey: String?,
        responseType: Class<T>,
        onError: BillingErrorCode,
    ): T =
        try {
            restClient.post()
                .uri(properties.baseUrl + path)
                .headers { h ->
                    h.set(HttpHeaders.AUTHORIZATION, authHeader)
                    h.contentType = MediaType.APPLICATION_JSON
                    if (idempotencyKey != null) h.set("Idempotency-Key", idempotencyKey)
                }
                .body(body)
                .retrieve()
                .body(responseType) ?: throw AppException(onError)
        } catch (e: RestClientResponseException) {
            // 토스가 코드/메시지를 본문에 담아줌 — 사유는 로깅만(사용자 메시지는 기본값)
            log.warn("토스 빌링 호출 실패 path={} status={} body={}", path, e.statusCode, e.responseBodyAsString)
            throw AppException(onError, cause = e)
        }

    @JsonIgnoreProperties(ignoreUnknown = true)
    data class IssueResponse(val billingKey: String, val card: Card? = null)

    @JsonIgnoreProperties(ignoreUnknown = true)
    data class Card(val company: String? = null, val number: String? = null, val cardType: String? = null)

    @JsonIgnoreProperties(ignoreUnknown = true)
    data class ApproveResponse(val paymentKey: String, val orderId: String, val approvedAt: String? = null)
}

data class IssuedBilling(val billingKey: String, val cardCompany: String?, val cardNumber: String?, val cardType: String?)
data class ApprovedPayment(val paymentKey: String, val orderId: String, val approvedAt: String?)
```

> 주의: `AppException(onError, cause = e)`는 `AppException(errorCode, message=defaultMessage, cause)` 시그니처를 사용한다. named arg `cause`가 두 번째 위치 message를 건너뛰므로 `AppException(errorCode = onError, cause = e)`로 호출하거나 message 기본값을 그대로 둔다. 컴파일 에러 시 `AppException(onError, onError.defaultMessage, e)`로 명시.
