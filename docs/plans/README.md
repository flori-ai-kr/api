# 구현 계획 (Plans)

기능·리팩토링의 구현 설계 문서를 모은다. 착수 전 합의한 범위·접근·단계를 기록하고, 완료 후에도 "왜 이렇게 구현했는가"의 근거로 남긴다.

## 작성 규칙

- **파일명**: `yy-mm-dd-{슬러그}.md` (2자리 연도, 예: `26-06-01-reservation-conversion.md`).
- **언어**: 한국어(코드/식별자만 영어).
- 계획이 바뀌면 문서를 갱신하고 상단 날짜를 최신화한다.
